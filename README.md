# 💊 STM32 Smart Pill Dispenser

> STM32 기반 자동 약 디스펜서 — 상태 머신 컨베이어 제어, 7일 Auto-Load, BLE 통신, RTC 알람을 통합한 임베디드 시스템

---

## 📌 프로젝트 개요

본 프로젝트는 STM32 마이크로컨트롤러를 중심으로 설계된 **지능형 자동 약 공급 시스템**입니다.  
사용자는 블루투스 앱을 통해 7일치 복약 일정을 전송하고, 지정된 시각에 RTC 알람이 트리거되어 해당 요일의 약이 자동으로 공급됩니다.  
컨베이어(DC 모터)와 호퍼(스텝모터)의 동작은 초음파 센서 피드백과 상태 머신으로 정밀하게 제어됩니다.

## ⚙️ 핵심 기능

### ① 상태 머신 기반 컨베이어 제어

컨베이어 동작은 5단계 FSM(Finite State Machine)으로 관리됩니다.

```
STATE_IDLE → FORWARD → WAIT → BACKWARD → FINISHED
```

| 상태 | 동작 |
|------|------|
| `STATE_IDLE` | 초기 대기 — 명령 수신 시 FORWARD 전이 |
| `FORWARD` | DC 모터 정방향 구동. 초음파 센서 거리 ≤ 1cm 감지 시 모터 정지 + 부저 ON |
| `WAIT` | 외부 버튼(PB5) 입력 대기 → 확인 후 역방향(CCW) 전환 |
| `BACKWARD` | 역방향 구동. 거리 ≥ 15cm 도달 시 정지 → 슬롯 회전 + 약 보충 실행 |
| `FINISHED` | 시퀀스 완료, IDLE 복귀 |

**핵심 설계 포인트**
- 초음파 센서로 물리적 종단을 감지해 소프트웨어 엔코더 없이 정위치 제어
- 부저를 사용자 피드백 수단으로 활용 (약 배출 완료 알림)
- 버튼 인터럽트와 폴링을 혼합하여 WAIT 상태의 응답성 확보

### ② 7일치 Auto-Load 시퀀스

블루투스로 수신한 **7자리 비트열** 기반으로 요일별 약 공급을 자동화합니다.

**명령 포맷**
```
L1010100   ← 'L' 접두어 + 7자리 비트 (월~일)
 │││││││
 월화수목금토일
```

**처리 흐름**
```
수신 비트열 파싱
     │
     ├─ '1' → Supply_Pill() 호출
     │         └─ Stepper2 구동 → 호퍼에서 약 하강
     │
     └─ '0' → SKIP (LCD2에 "SKIP" 표시)
     
매 요일마다:
  Stepper2_One_Day() → 메인 약통 1칸 회전 → 500ms 안정 대기
```

**LCD2 실시간 상태 표시**

| 슬롯 상태 | LCD 출력 |
|-----------|----------|
| 약 공급 중 | `DAY3: LOADING` |
| 해당일 없음 | `DAY3: SKIP` |

### ③ UART 기반 블루투스 통신 프로토콜

| 채널 | UART | 속도 | 용도 |
|------|------|------|------|
| HC-06 BLE | UART1 | 9600 bps | 앱 ↔ MCU 명령 수신 |
| PC 디버그 | UART2 | 115200 bps | TeraTerm 시리얼 모니터 |

**인터럽트 기반 수신 구조**
```
UART1 RX Interrupt
    │
    ▼
uart2_buffer[]에 바이트 누적
    │
    ▼
'\n' 감지 시 → 명령 완료 플래그 세트
    │
    ▼
Main Loop에서 명령 파싱 처리
```

**지원 명령어**

| 명령 | 포맷 예시 | 동작 |
|------|-----------|------|
| 시간 동기화 | `T2025/05/29 14:30:00` | RTC에 현재 시각 설정 |
| 알람 설정 | `A08:00` | RTC ALRMAR에 복약 시각 등록 |
| Auto-Load | `L1010100` | 7일 약 적재 시퀀스 시작 |
| 서보 제어 | `servo` | 배출 서보 수동 트리거 |
| 스텝 제어 | `step` | 슬롯 스텝모터 1칸 수동 이동 |

### ④ RTC 알람 및 인터럽트 처리

**알람 설정 흐름**
```
앱 → 'A' 명령 수신
    │
    ▼
시각 파싱 → BCD 변환
    │
    ▼
STM32 RTC ALRMAR 레지스터 직접 설정
    │
    ▼
EXTI Line 17 (RTC Alarm) 인터럽트 활성화
```

**알람 처리**
```c
// 알람 ISR
void RTC_Alarm_IRQHandler(void) {
    pill_alarm_flag = 1;   // 플래그 세트
    // 실제 처리는 Main Loop에서 폴링
}

// Main Loop
if (pill_alarm_flag) {
    pill_alarm_flag = 0;
    Auto_Load_Sequence();
}
```

**NVIC 인터럽트 우선순위**

| 우선순위 | 인터럽트 소스 | 이유 |
|----------|---------------|------|
| 0 (최고) | 타이머 (TIM) | 모터 PWM 타이밍 정확도 |
| 1 | 외부 버튼 (EXTI) | 사용자 응답성 |
| 2 | UART RX | 명령 유실 방지 |

**RTC 정밀도**
- LSE(32.768 kHz) 외부 크리스탈 사용
- 캘리브레이션 루틴 및 오차 보정 알고리즘 적용
- **30일 기준 누적 오차 1분 이내** 달성

## 🚀 빌드 및 실행

**개발 환경**
- IDE: STM32CubeIDE
- HAL 라이브러리: STM32CubeF4
- 디버그: ST-Link V2 + TeraTerm (115200 bps)

**블루투스 앱 명령 순서 (초기 설정)**
```
1. T2025/05/29 14:30:00   ← 시각 동기화
2. A08:00                  ← 복약 알람 설정
3. L1010100                ← 7일 적재 시퀀스 시작
```

## 📝 개발 메모

- `pill_alarm_flag` 폴링 방식 채택 이유: ISR 내 모터 제어 시 타이밍 충돌 방지
